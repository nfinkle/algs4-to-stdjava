PriorityQueue (Javadoc API) has all of the same operations as MinPQ (Javadoc API).

The main difference between PriorityQueue and MinPQ is that the comparator in the constructors with a comparator for PriorityQueue must be of type Comparator<? super Key>, while the comparator in the constructors with a comparator for MinPQ are of type Comparator<Key>. The Comparator<? super Key> means that the comparator must be able to compare two objects of type Key and of any type from which Key inherits. For example a Comparator<? super Queue<T>> must accept arguments of the type Iterable<T> and Object because Queue<T> inherits from those interfaces.

algs4 also contains the priority queue IndexMinPQ (Javadoc API) which features the ability to reference an individual key to modify. There is currently no equivalent in the standard Java libraries.
